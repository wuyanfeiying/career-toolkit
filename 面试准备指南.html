我理解 Vibe Coding 的核心是把"写代码"这件事从"手工实现"转变为"需求描述+AI生成+人工审查"。

关键不是AI写了多少代码，而是人能不能准确描述需求、审查质量、把控方向。

2. 任务拆解：把大需求拆成AI能理解的小任务
3. Prompt + 上下文管理：我会写CLAUDE.md给AI提供项目上下文，不是每次从零开始对话
4. 生成+审查：AI生成代码后我会看一遍，理解逻辑，有问题通过看报错信息定位后让AI修正
5. 质量把控：最终确保代码能跑、逻辑对、可维护

和传统开发的区别是：传统是我写每一行，Vibe Coding是我做架构师+PM+code reviewer，AI做实现。效率可以提升几倍，但前提是人要有业务理解力和技术判断力。

🟡 高概率追问

高概率 MCP 协议相关
你简历里提到了MCP协议，能讲讲你的理解吗？
诚实但有深度的回答：

MCP（Model Context Protocol）是 Anthropic 提出的一个开放协议，本质上是给大模型定义了一套标准化的工具调用接口，类似于AI世界的USB-C。

它解决的问题是：不同的AI模型要调用外部工具（搜索、数据库查询、代码执行等），以前每家都有自己的接口格式，MCP统一了这个标准。

在我们公司的场景里，MCP被用来封装公式执行服务——AI Agent需要调用runMultiFormula来计算金融指标，通过MCP协议把这个能力暴露给模型。

我主要是在卡片工程和Skill评估的工作中接触到MCP的，了解它的协议结构和使用场景。具体的MCP服务端实现不是我写的，但我理解它的工作原理和在系统中的角色。
⚠️ 注意：不要说"MCP服务端是我开发的"。如果面试官追问实现细节（比如JSON-RPC、transport层），诚实说"这部分具体实现不是我负责的，我主要是作为使用方理解协议规范"。
