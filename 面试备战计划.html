1️⃣ 搞懂 MCP 协议（不能再说"AI做的不了解"）

任务 1.1：阅读 MCP 官方文档
⏱ 2小时 决定成败
目标：面试时能用自己的话讲清楚 MCP 是什么、解决什么问题、怎么工作。

阅读路径（按顺序）：

  • MCP 官网介绍 — 理解"AI世界的USB-C"比喻
  • 架构概念 — Host / Client / Server 三角关系
  • Tools 概念 — 最重要，你公司就是用这个
  • Resources 概念 — 了解即可

读完后你需要能回答：

  • MCP 是什么？一句话描述
  • MCP Host、Client、Server 分别是什么角色？
  • Tool 和 Resource 的区别是什么？
  • MCP 的传输层有哪两种？(stdio / HTTP+SSE)
  • 你公司的 gpt_server 里 MCP 是怎么用的？（公式执行服务）
任务 1.2：动手做一个最小 MCP Server
⏱ 3小时 决定成败
为什么要动手：面试官一追问"你做过MCP吗"，你可以说"我自己写过一个MCP Server来理解协议，虽然不复杂但帮我理解了整个流程"。这比"我读过文档"强10倍。

用 Vibe Coding 做！这本身就是展示你工作方式的案例

用 Claude Code 或 Cursor，让AI帮你写一个简单的 MCP Server：

// 示例目标：一个天气查询 MCP Server // 接入 MCP 协议，暴露一个 tool: get_weather(city) // 让 Claude Desktop / Cursor 能调用它 // 技术栈：@modelcontextprotocol/sdk + Node.js // 参考：https://modelcontextprotocol.io/quickstart/server npm init -y npm install @modelcontextprotocol/sdk
关键：虽然代码是AI帮你写的，但你要理解每一行在干什么。重点关注：
  • Server 怎么注册 Tool 的（tool name, description, input schema）
  • Tool handler 怎么返回结果的
  • Client 怎么发现和调用 Tool 的
  • 跑通一个最小 MCP Server（哪怕是 hello world）
  • 能在 Claude Desktop 或 Cursor 里调用你的 Server
  • 能口述整个 MCP 调用流程（Client → Server → Tool → Response）
  • 把代码推到 GitHub（面试时可以展示）
任务 1.3：回到 gpt_server 项目，理解公司的 MCP 实现
⏱ 1.5小时 加分

你已经自己做了一个 MCP Demo，现在回去看公司项目就容易多了：

  • 打开 gpt_server/src/，找到 MCP 相关路由
  • 看看 /gptAgent/mcp 这个路由做了什么
  • 理解 runMultiFormula 这个 Tool 是怎么注册的
  • 公式依赖解析和并发执行的逻辑是什么
面试话术准备："我们公司用 MCP 封装了一个公式执行服务，核心 Tool 叫 runMultiFormula——它支持批量执行多个金融公式，会自动解析公式之间的依赖关系，同一依赖层级的公式并发执行，有依赖的串行保序。这个服务端不是我从头开发的，但我为了深入理解协议，自己也做了一个 MCP Demo。"
  • 能说清楚公司 MCP 服务的功能（runMultiFormula）
  • 能说清楚公式依赖解析的思路

2️⃣ 练习讲项目故事（你说"说不出来"，必须解决）

⏱ 3小时 决定成败
STAR 法则：Situation（背景）→ Task（任务）→ Action（你做了什么）→ Result（结果）
每个故事控制在 2 分钟以内。

S：公司的AI Agent要准确回答投研问题，需要结构化的知识体系支撑

A：①梳理了金融业务公式链 ②设计了卡片结构和字段规范 ③开发了5类审计工具 ④打通了MongoDB和向量库的同步链路
R：累计产出50+张卡片，建立的SOP和审计规则被团队沿用

故事 2：Skill 评估框架

S：团队在迭代AI Skill时没有系统化的评估手段，改完之后不知道是变好还是变差

S：甲方需要一个智能资产评级系统的原型来验证需求，时间紧

任务 2.2：准备"最复杂的代码"这个问题
⏱ 1小时 决定成败

你自评"说不出来"，但其实你有东西可以讲。选一个你真正懂的：
